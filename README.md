## Overview
This project simulates the responsibilities of an **Anti-Fraud Data Analyst** by using **SQL** to detect suspicious patterns in e-commerce transactions.

## Objectives
- Use SQL to analyze e-commerce transaction data for anomalies  
- Identify patterns such as refund abuse, repeated high-value transactions, and cross-country activity  
- Create a simple fraud scoring model using conditional logic  
- Summarize findings in a report with visuals (Obsidian + PDF)
